# 01-organized-docs

This folder contains all key info, context, and change log documents for the project, organized for systematic tracking and onboarding.

## Structure
- 01-CHANGELOG.md — Centralized change log
- 02-PROJECT_CONTEXT_AND_SCOPE.md — Project context and scope
- 03-ROADMAP.md — Roadmap and budget
- 04-RELEASE_HANDBOOK.md — Release and deployment
- 05-SESSION_LOGS/ — Meeting/session notes
- 06-ARCHIVE/ — Old/legacy docs

## Usage
- Add new docs here with a clear number prefix.
- Update the changelog with every significant change.
- Move all relevant docs from scattered locations here for easy access.
